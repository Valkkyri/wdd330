Chapter 2 - Programming Basics

Write comments in your code! You'll be grateful you did when you have to revisit an old project. Keep it simple and easy to understand. Comments make a master Ninja.

  • A Statement is a single line of code terminated by a semicolon;
  • "A block is a series of statements that are collected together inside curly braces."
    {
    // this is a block containing 2 statements
    const message = 'Hello!';
    alert(message);
    }

There 7 type different data types, 6 of which are primitive. Primitive Types: String, Symbol (The symbol primitive data type was only introduced in ES6.), Number, Boolean, Undefined, Null. The 7th data type is an object, which can be several different types of data... (just not the primitive ones). Basically, if it contains more than a single value, it's an object (think arrays, key/value pairs, etc.).

Use Const as often as possible, to make your code easier to read and to debug (sad day, I like let better for some reason. So silly).

Global Scope (whole document) and Local Scope (only inside the block), keep the number of global variables to a minimum.

Properties tell us about the object or value; Methods allows us to manipulate the object or value. You can tell the difference between Properties and Methods by the (parentheses at the end of the Method). EX:
properties.length == property
variable.method() == method.
